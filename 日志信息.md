# 开发日志 2026.5.13

### **Player 组件全面解耦**
- 解耦 `PlayerController`，改为独立 `Node`，挂载于 `Player` 根节点下，不再内嵌于 `Player` 类。
- 远程玩家对象通过 `NetSyncBase.IsOwner` 判断，非所有者不初始化控制器、UI 及物理处理。
- 状态机、视觉模型等子组件均解除对 `Player` 基类的直接依赖，只依赖根节点 `Node3D`。

### **旋转类型统一为 Vector3（欧拉角）**
- 彻底清理项目中所有 `Quaternion` 残留。
- `NetObject.Rotation` 字段、所有 RPC 方法参数（`Rpc_HostSyncRegister`、`Rpc_ReportToServer`、`Rpc_SyncToPeer`、`Rpc_SendTransform`）以及 `NetObjectManager` 设置旋转处均改为 `Vector3`。
- `RegisterObject`、`SpawnObject` 等调用点全部传入欧拉角，编译 0 错误。

### **日志系统 CatLog**
- 新建静态工具类 `CatLog`，提供分级日志：`Info`、`Ok`、`Warn`、`Err`、`Debug`。
- 支持按级别颜色输出与开关控制，已接入网络核心流程，替代零散 `GD.Print`。

## 1：联机框架全面重构与完成（5月5日 – 5月12日）

### 场景与交互网络化（D2/D4）——基本完成，待收尾
- **场景管理**：预制体补发与客户端生成管道已调通，客户端玩家加入时主机通过 `Rpc_SyncToPeer` 补发，客户端正确创建实例。
- **断线清理**：已规划，待实现（当前 `OnPeerDisconnected` 中已预留日志，详细逻辑在下一步行动清单首位）。
- **交互网络化**：尚未启动，后续将按攻击 → 物品拾取 → 容器操作的顺序逐步 RPC 化。

### 玩家与实体同步（D3）——全部完成
- **`NetTransformSync`**：
  - 位置使用根节点，旋转支持分离的 `_rotMesh`（`m_PlayerModel`），不干扰摄像机。
  - 主机与客户端均实现双向上报与转发，主机在收到客户端上报时先更新本地镜像，再排除发送者转发给其他客户端。
  - 非 Owner 使用 `Lerp` 平滑插值，频率 20Hz，流畅无抖动。
- **`NetAnimationSync`**：
  - 通过同步 `StateMachine` 的整数值（`PlayerState` 枚举）驱动远程 `AnimationTree`。
  - 主机/客户端双向机制同上：客户端上报状态到主机，主机更新本地 `StateMachine` 并转发给其他客户端。
- **双客户端验证通过**：主机与任意客户端移动、攻击、跳跃，所有端均能实时看到位置、旋转、动画的正确同步。
- **所有权隔离**：`PlayerController` 入口添加 `if (player.m_PlayerData == null) return;`，杜绝远程角色被本地输入控制。

### 网络基础层（D1）——全部完成
- **`NetCore` 成型**：合并连接管理、LAN 广播发现（UDP）、ENet 连接、RPC 路由；提供 `ServerID = 1`、`LocalPeerID` 等核心属性。
- **`NetID` 与 `NetObject`**：分布式对象标识由 `UserID + 自增序号` 构成；`NetObject` 包含位置、旋转（最终统一为 `Vector3` 欧拉角）、预制体哈希、Owner PeerID。
- **对象注册与生成管道**：
  - `NetObjectRegistry` 单例管理所有 `NetObj`，主机分配 ID 并广播 `Rpc_HostSyncRegister`。
  - 客户端上报 `Rpc_ReportToServer`，主机再广播。
  - 新客户端加入补发全部现有对象（`Rpc_SyncToPeer`）。
- **`NetObjectManager`**：作为唯一实例化入口，维护预制体字典 `m_PrefabDict`，监听 `OnSpawned` 事件自动创建节点并挂载 `NetSyncBase`。
- **`NetSyncBase`**：挂载于每个网络节点，持有 `NetObject`，提供 `IsOwner` 判断，无信号设计。
- **远程屏蔽**：`Player._Ready` 中若 `m_PlayerData == null`（远程玩家）则关闭 `_Process`/`_PhysicsProcess`，不启用控制器、UI、摄像机。
- **旋转类型统一**：全链路将 `Quaternion` 旋转改为 `Vector3` 欧拉角，确保与 `GlobalRotation` 一致，同步更直接。

## 2：玩家状态机与战斗（5月初）

- **玩家状态机正式上线**：`StateMachine` 节点独立，定义 Idle/Walk/Jump/Fall/Attack 等状态，基于速度和地面状态自动切换。
- **攻击闭环**：通过 `AnimationNodeOneShot` 触发攻击动画，关键帧调用 `PerformAttack()` 射线命中并造成伤害，攻击期间锁定移动。
- **树木受击**：耐久通过 `CustomData["durability"]` 管理，避免污染静态 Resource。
- **状态机与移动解耦**：`PlayerController` 中增加攻击状态判断，攻击时禁用移动但不影响重力。

---

## 3：实体与 AI 基础（4月底 – 5月初）

- **Player 重构**：`PlayerUIHandler` 接管所有 UI 管理，`Player` 仅保留交互路由，减少约 80 行代码，为网络层接入铺路。
- **NPC 继承统一**：废弃旧 `NpcBase`，`Npc : Humanoid` 完全融入实体体系，动画树自动响应移动状态。
- **巡逻与导航**：纯直线巡逻系统落地（极坐标随机选点，可配置参数）；导航网格全局绘制并烘焙，预留 `NavigationAgent3D` 寻路接口。
- **视线与追击**：`m_eye` 升级为 `RayCast3D`，`See()` 检测触发巡逻→追击切换，目标丢失后自动回退。
- **状态机雏形**：Npc  Patrol/Chase 状态调度，`_PhysicsProcess` 极简化为四步流程。

## 4：物品与背包系统（4月）

- **物品系统数据驱动**：`ItemComp` 仅持有 `[Export] ItemData`，物品属性完全配置化；`ItemData` 为可编辑 `Resource`，支持中文分组。
- **背包核心逻辑**：`SlotComp` + `InventoryComp` 实现格子自治、堆叠、拖拽交换、拆分，严格遵循数据/表现分离。
- **背包持久化闭环**：读档自动实例化独立格子，回填存档数据；`RefSlot()` 实时同步存档，实现“操作即存档”。
- **交互优化**：移除旧版检测区域，统一使用 `m_eye`（RayCast3D）进行射线拾取、攻击检测。
- **放置物与容器**：建立 `PlacedBase` 继承体系，`PlacedData` 定义类型枚举；`ContainerComp` 实现 `IInventoryHolder`，支持 E 键开关、跨库存拖拽、持久化容器物品。
- **容器 UI 完善**：复用 `InventoryComp` 双面板，支持拖拽转移与同库存交换，预留过滤器接口。

## 5：架构与框架奠基（3月 – 4月初）

- **基础架构确立**：搭建 `GameCore`、`NetworkCore`（早期ENet P2P）、`UIPanelBase`，确立单例管理器、无全局静态、服务定位器模式。
- **实体继承体系**：建立 `CreatureBase → Humanoid → Player` 以及 `CreatureBase → Humanoid → Npc` 双线，统一移动、重力、动画树、视线等基础设施。
- **数据表现分离铁律**：移除 `PlayerData` 对场景节点的直接引用，确保存档安全、可网络同步；引入 `PlayerManager` 统一管理玩家实例。
- **资源管理器**：实现按脚本类型自动分类加载预制体，支持 `Player`、`ItemComp`、UI 面板等快速获取。
- **存档系统**：`SaveData` 采用 `Dictionary<int, PlayerData>` 结构，支持多角色选择、增删存档；`PlayerData` 继承 `Resource`，内置背包数据，全程深拷贝保护。
- **初始化流程**：解决 `AddChild(null)`、`private set` 编译错误等时序问题，确保所有管理器 `Init()` 显式调用，依赖顺序可控。
