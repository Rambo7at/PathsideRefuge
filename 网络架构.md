# 网络同步架构总览

## 核心架构映射

| 本项目命名 | 对应 Valheim 概念 | 类型 | 职责 |
| :--- | :--- | :--- | :--- |
| **NetCore** | **ZNet + ZDOMan** | Autoload (单例) | 网络中枢：连接管理 / RPC / 分布式对象同步 |
| **NetSyncData** | **ZDO** | Class (数据类) | 核心数据容器，存储需要同步的状态 |
| **NetSyncBase** | **ZNetView** | Node (组件) | 挂载在物体上的网络视图接口 |
| **NetTransformSync** | **ZSyncTransform** | Node (组件) | 专门同步位置与旋转 |
| **NetAnimationSync** | **ZSyncAnimation** | Node (组件) | 专门同步动画状态 |

---

## 1. NetCore (Autoload – 简化后的唯一中枢)

**原 Valheim 中 `ZNet` 和 `ZDOMan` 被合并为本项目的 `NetCore`。**

**职责：**
- 管理所有 `NetSyncData` 实例 (原 `ZDOMan` 职责)
- 维护每个 Peer 的同步状态
- 周期性收集“脏数据” (Dirty Data)，并按需定向/广播发送
- 同时负责底层的连接管理、UDP 发现和 RPC 路由 (原 `ZNet` 职责)

---

## 2. NetSyncData

**职责：**
- 存储一个网络物体的所有同步状态
- 拥有唯一标识符 `ID`
- 处理数据版本号 (Revision) 以检测变化

**对应：** `ZDO`

**字段结构：**
*   **ID**: 唯一标识符 (类似 `ZDOID`)
*   **Owner**: 拥有者 Peer ID
*   **DataRevision**: 数据版本号 (用于检测整体变化)
*   **OwnerRevision**: 拥有者版本号
*   **Position**: 位置
*   **Rotation**: 旋转
*   **PrefabHash**: 预制体哈希 (用于生成)
*   **ExtraData**: `Dictionary<int, object>` (按类型分桶存储，可序列化)

---

## 3. NetSyncBase (Node)

**职责：**
- 挂载到预制体根节点
- 持有 `NetSyncData` 的引用
- 提供 RPC 注册/调用封装
- 提供 `IsOwner` / `ClaimOwnership` 等所有权接口

**对应：** `ZNetView`

---

## 4. NetTransformSync (Node)

**职责：**
- 依赖 `NetSyncBase`
- **如果是 Owner**：写入 `Data` -> 增加 `DataRevision`
- **如果不是 Owner**：从 `Data` 读取 -> 执行平滑插值 (Interpolation)

**对应：** `ZSyncTransform`

---

## 5. NetAnimationSync (Node)

**职责：**
- 同步动画状态机参数、触发器等
- 数据通过 `NetSyncData.ExtraData` 或直接 RPC 进行同步

**对应：** `ZSyncAnimation`
