# 🎮 途畔归所 · 个人精简版开发计划书

**版本：** 2.1  
**目标：** 单人完成 1-4 人联机 RPG 的最小可玩闭环（探索→战斗→获取装备→变强）  
**架构原则：** 主机权威，客户端表现，朋友联机不设强反作弊，仅保证数据一致性与稳定性  
**整体进度：** ⬜⬜⬜⬜ 0%✅

---

## 📋 精简开发计划

### 🔩 阶段一：角色闭环（1-2 周）[⬜]
**目标：** 玩家拥有属性、装备栏，装备影响战斗力，能看到自己的面板。

| ID | 任务 | 要点 | 预计人日 | 进度 |
|----|------|------|----------|------|
| 1.1 | 属性系统 | 基础属性（力量/敏捷/体质等），派生属性（攻击力/生命上限）<br>接入 PlayerData，主机维护 | 1.5 | ⬜ |
| 1.2 | 装备栏 UI | 新增装备槽容器（复用 InventoryComp），绑定 6-8 个固定槽位<br>拖拽装备/卸下 | 1.5 | ⬜ |
| 1.3 | 装备效果生效 | 穿戴武器→影响攻击力，防具→影响生命/防御<br>主机验证穿戴请求并广播 | 1.5 | ⬜ |
| 1.4 | 角色面板 | 显示属性、装备、背包（基于 PlayerGUI 扩展） | 1 | ⬜ |
| | **小计** | | **5.5 人日** | ⬜ |

> 🚩 里程碑 M1：本地换装，属性变化，面板显示正确。 [⬜]

---

### ⚔️ 阶段二：战斗验证（1-2 周）[⬜]
**前置验证：** 先做"砍树"，再做"打怪"。  
利用树木（静态 NetObject）验证伤害公式、RPC 结算、血量同步框架。验证通过后，将同一套框架迁移至 NPC，额外处理 AI 行为同步与反击逻辑。

| ID | 任务 | 要点 | 预计人日 | 进度 |
|----|------|------|----------|------|
| 2.1 | 伤害框架验证（砍树） | 玩家攻击树木→RPC 请求主机→主机结算伤害并广播<br>树木血量归零后倒下，生成掉落物（与阶段三 3.1 联动） | 2 | ⬜ |
| 2.2 | 伤害公式化 | 伤害 = 基础攻击力 + 武器攻击力 × 系数<br>防御/减伤接入属性 | 0.5 | ⬜ |
| 2.3 | NPC 血量同步与受击 | NPC 挂 NetSyncBase，同步血量、当前目标<br>主机 AI 驱动，客户端表现受击动画 | 1.5 | ⬜ |
| 2.4 | NPC 死亡与掉落 | 主机广播死亡，所有客户端播放死亡动画<br>主机根据掉落表生成网络物品实体（复用 NetObjectManager） | 1.5 | ⬜ |
| | **小计** | | **5.5 人日** | ⬜ |

> 🚩 里程碑 M2：可联机砍树，树木倒下掉落资源；可联机打 NPC，装备差异影响伤害，NPC 死亡后掉落物品。 [⬜]

## ❌ 6. 明确不做的功能（当前版本）

- ❌ 远程攻击 / 技能系统 / Buff 状态效果（需属性系统稳定后再扩展）
- ❌ 纸娃娃系统（装备仅属性影响，外观先用基础模型）
- ❌ 世界流式加载（使用单场景大地图 + 距离裁剪）
- ❌ 断线重连 / 主机迁移（主机离开即解散房间）
- ❌ 队伍系统、好友列表、私聊（聊天先做广播）
- ❌ 小地图全貌（先做队友标记）
