# Godot 3D 可复用联机游戏框架 —— 开发计划书
## 核心定位
- 目标：构建**通用、可跨项目复用、支持 P2P 联机**的 3D 独立游戏底层框架
- 适用类型：生存、冒险、RPG、建造、小型联机游戏
- 架构风格：参考 Valheim 标准架构，**控制层 / 表现层 / 数据层 彻底分离**
- 核心特性：模块化、低耦合、可扩展、编辑器友好、适合快速开发

---

# 一、Game Core（游戏核心框架）
## 目标：提供全局唯一入口、统一生命周期、管理器调度
- ✅ Autoload 全局单例系统
- ✅ 多 Manager 统一注册与初始化
- ✅ 游戏全局生命周期管理
- ✅ 工具类库（ToolUtils）
- ✅ 控制台系统（ConsoleSystem）
  - 字典+委托命令路由
  - 支持命令+参数解析
  - 支持调试、生成、测试指令

---

# 二、Network Core（P2P 联机核心）
## 目标：实现稳定低延迟局域网联机，支持多人同屏
- ✅ ENet 底层 P2P 连接
- ✅ 主机/客户端模式
- ✅ 局域网 / 虚拟局域网联机
- ✅ RPC 调用与 Resource 传输
- 🔄 玩家位置同步
- 🔄 玩家外观/装备同步
- 🔄 延迟补偿
- 🔄 掉线重连
- 🔄 联机房间系统

- ✅ Player 角色实体

- 🔄 怪物 AI / 仇恨 / 行为树

# 框架开发路线（优先级从高到低）
1. 完善存档系统（SaveManager）
2. 完善网络同步（位置/外观）
3. 实体状态机 + 属性系统
4. 游戏内 HUD / 暂停 UI
5. 背包系统 + 物品扩展
6. 联机房间 UI
7. 交互系统扩展
8. AI 与怪物系统
9. 工具链与资源自动化

---

# 框架最终目标
- 可直接用于制作**联机 3D 独立游戏**
- 无技术债务
- 结构清晰、易于扩展
- 可跨项目复用
- 可安全推送到 GitHub 作为独立框架
